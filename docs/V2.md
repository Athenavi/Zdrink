## 🚀 Next.js 方案的核心优势

### 1. API 处理：与 Django 无缝集成

**现成方案：next-drf**

这是一个专门为 Next.js + Django REST Framework 设计的全栈框架，提供了：

- 预配置的用户认证和 API 集成
- 支持 AWS Cognito、Firebase、Auth0 等多种认证方式
- CLI 工具一键初始化项目
- 前后端分离但统一管理

```bash
# 一键创建 Next.js + Django DRF 项目
npx next-drf@latest

# 同时启动前后端
npm run dev  # 前端 :3000, 后端 :8000
```

**认证方案：NextAuth + Djoser**

社区有成熟的 JWT 认证集成方案：

- 使用 NextAuth 的 credentials provider 调用 Djoser 的 `/auth/jwt/create/` 端点
- 在 session 回调中存储 access/refresh token
- 使用 axios 拦截器自动附加 token 到请求头
- 实现自动 token 刷新逻辑

### 2. 多租户架构：基于子域名的优雅实现

Next.js 完全支持基于子域名的多租户方案，这正是你的 Django 后端所需要的：

**核心实现思路**：

```
https://restaurant-a.yourdomain.com  → 租户A
https://restaurant-b.yourdomain.com  → 租户B
```

**实现步骤**：

```typescript
// middleware.ts - 核心租户识别
import {NextRequest, NextResponse} from 'next/server';

export function middleware(req: NextRequest) {
    const host = req.headers.get('host') || '';
    const subdomain = host.split('.')[0];  // 提取子域名

    // 将租户信息注入请求头，供 API 调用使用
    const requestHeaders = new Headers(req.headers);
    requestHeaders.set('x-tenant', subdomain);

    return NextResponse.next({
        request: {headers: requestHeaders}
    });
}

export const config = {
    matcher: ['/((?!api|_next/static|_next/image|favicon.ico).*)'],
};
```

```typescript
// 在 API 调用中自动带上租户信息
// lib/api.ts
export async function apiClient(endpoint: string, options?: RequestInit) {
    const tenant = document.cookie
        .split('; ')
        .find(row => row.startsWith('x-tenant='))
        ?.split('=')[1];

    return fetch(`${process.env.NEXT_PUBLIC_API_URL}${endpoint}`, {
        ...options,
        headers: {
            'Content-Type': 'application/json',
            'X-Tenant': tenant || '',
            ...options?.headers,
        },
    });
}
```

### 3. 跨平台：一套代码，三端运行

虽然 Next.js 本身是 Web 框架，但可以配合打包工具实现真正的跨平台：

**开源模板：template-next-cross**

这是一个基于 Next.js + Electron + Capacitor 的跨平台模板：

- **Web 端**：Next.js 原生构建
- **桌面端**：Electron 打包，支持 Windows/macOS/Linux
- **移动端**：Capacitor 转换，支持 iOS/Android

**技术架构**：

```
┌─────────────────────────────────────────────┐
│              Next.js App                     │
│         (一套 React 代码)                     │
└─────────────────────────────────────────────┘
        │              │              │
        ▼              ▼              ▼
   next build    Electron      Capacitor
        │              │              │
        ▼              ▼              ▼
     Web 端        桌面端          移动端
    (浏览器)    (原生应用)      (iOS/Android)
```

**Capacitor 集成配置**：

```bash
# 构建 Next.js 静态导出
npm run build  # 配置 output: 'export'

# 添加 Capacitor
npx @capacitor/cli init
npx cap add android
npx cap add ios

# 同步构建产物
npx cap copy
npx cap open android
```

### 4. 响应式设计：Tailwind CSS + 移动端适配

Next.js 默认支持 Tailwind CSS，配合移动端适配方案效果极佳：

**核心适配方案**：

```tsx
// app/layout.tsx - 在 head 中注入动态根字体大小
export default function RootLayout({children}) {
    return (
        <html lang="zh">
        <head>
            <script
                dangerouslySetInnerHTML={{
                    __html: `
              (function() {
                const UI_WIDTH = 375;
                const baseFontSize = 16;
                function setRootFontSize() {
                  const width = document.documentElement?.clientWidth;
                  const fontSize = ((width / UI_WIDTH) * baseFontSize).toFixed(4);
                  document.documentElement.style.fontSize = fontSize + 'px';
                }
                setRootFontSize();
                window.addEventListener('resize', setRootFontSize);
              })();
            `,
                }}
            />
        </head>
        <body>{children}</body>
        </html>
    );
}
```

**响应式组件示例**：

```tsx
// 使用 Tailwind 响应式类
export function ResponsiveLayout() {
    return (
        <div className="px-4 md:px-8 lg:px-16">
            {/* 移动端: 垂直排列, 桌面端: 网格布局 */}
            <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4">
                <div className="text-sm md:text-base">菜单项</div>
            </div>

            {/* 移动端隐藏, 桌面端显示 */}
            <aside className="hidden lg:block">桌面侧边栏</aside>

            {/* 移动端底部导航 */}
            <nav className="fixed bottom-0 left-0 right-0 bg-white md:hidden">
                移动端菜单
            </nav>
        </div>
    );
}
```
